# Task Manager Deployment & Security Checklist

## Phase 1: Local Security Hardening (OWASP Top 10)
*Before pushing code to GitHub*

- [ ] **Protect Secrets:** Create a `.gitignore` file.
- [ ] **Ignore Files:** Add `.env` and `node_modules` to `.gitignore`.
- [ ] **Sanitize Input:** Install `express-mongo-sanitize` to prevent NoSQL Injection.
- [ ] **Secure Headers:** Install `helmet` to hide server information (e.g., `X-Powered-By`).
- [ ] **Rate Limiting:** Install `express-rate-limit` to prevent brute-force login attacks.
- [ ] **Audit Dependencies:** Run `npm audit fix` to resolve known vulnerabilities.

## Phase 2: Version Control (GitHub)
*Uploading the code safely*

- [ ] **Initialize Git:** Run `git init`.
- [ ] **Commit Code:** Run `git add .` and `git commit -m "Initial secure commit"`.
- [ ] **Create Repo:** Create a new empty repository on GitHub.
- [ ] **Push Code:** Add remote origin and push to the `main` branch.
- [ ] **Verify:** Check GitHub to ensure the `.env` file is **NOT** visible.

## Phase 3: Automated Security (Snyk)
*Continuous vulnerability scanning*

- [ ] **Install Snyk:** Install the "Snyk" app from the GitHub Marketplace.
- [ ] **Configure:** Select the Task Manager repository for monitoring.
- [ ] **Review:** Check the initial security report generated by Snyk.

## Phase 4: Cloud Deployment (AWS EC2)
*Going live*

- [ ] **Connect:** SSH into the EC2 instance (`ssh -i key.pem ubuntu@ip`).
- [ ] **Update Server:** Run `sudo apt update && sudo apt upgrade -y`.
- [ ] **Install Tools:** Install Node.js (v18+), NPM, Git, and PM2.
- [ ] **Clone Code:** Run `git clone <repo-url>`.
- [ ] **Install Dependencies:** Run `npm install` inside the project folder.
- [ ] **Set Secrets:** Create `.env` manually (`nano .env`) and paste DB URI/JWT Secret.
- [ ] **Start App:** Run `pm2 start index.js --name "task-manager"`.
- [ ] **Persistence:** Run `pm2 startup` and `pm2 save` so the app restarts on reboot.

## Phase 5: Networking & Access
*Exposing the app to the world*

- [ ] **AWS Firewall:** Edit Security Group Inbound Rules to allow Custom TCP (Port 3000).
- [ ] **Reverse Proxy (Optional):** Install Nginx to forward Port 80 to Port 3000.
- [ ] **Final Test:** Access the application via the Public IP in the browser.

## Phase 6: Future Improvements (CI/CD)
- [ ] **Automate:** Create a GitHub Action (`.yml`) to auto-deploy on push.
